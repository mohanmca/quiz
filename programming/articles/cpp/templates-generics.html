C++ Deep Dive — Templates, SFINAE, Concepts
Specialization, detection, constraints, variadics, folds, dependent names, CRTP, constexpr meta

1. Specialization & Overload

  • Function templates cannot be partially specialized — use overloads with constraints.
  • Class/alias/variable templates allow partial specialization.

2. SFINAE & Detection Idiom

  • Substitution failures remove candidates; do not error.
  • Use detection idiom or requires to test for expressions/types.

3. Concepts (C++20)

  • requires-clause clarifies intent and produces better diagnostics.
  • Constrain overloads to avoid ADL/ambiguity surprises.

4. Variadics & Fold Expressions

  • Packs and folds simplify reductions and forwarding.
  • Beware empty packs; provide identity elements.

5. Dependent Names

  • Use typename for dependent type names; template disambiguator for member templates.
  • Prefer this-> to access dependent base members.

6. CRTP & constexpr meta

  • CRTP provides static polymorphism/mixins; reduce virtual cost.
  • constexpr algorithms + type traits power compile-time decisions.
