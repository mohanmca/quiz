2. Binding & Collapsing Rules

  • T& binds only to lvalues; const T& binds to lvalue/xvalue/prvalue; T&& binds to rvalues (xvalue/prvalue).
  • Reference collapsing: & + & → &, & + && → &, && + & → &, && + && → &&.
  • Forwarding references: T&& in a template parameter with deduction, forward with std::forward(arg).

3. Materialization & Lifetime Extension

  • Materialization (C++17): prvalues materialize a temporary when a glvalue is needed (e.g., binding to const T&).
  • Lifetime extension: a temporary bound directly to a const reference lives to the reference’s scope.
  • No extension for a temporary bound to a local rvalue reference (int&&) variable.

4. Common Dangling Cases

  • Returning a reference to a local variable.
  • Capturing by reference in asynchronous callbacks.
  • Storing std::string_view to a temporary string.

5. Diagnostics & Heuristics

  • Prefer return-by-value; enable elision and moves. Avoid returning references to locals.
  • Add std::move at ownership transfer boundaries; not on local names you still need.
  • Use sanitizers (ASan/UBSan) and lifetime checkers (Clang) to catch misuse.
Tip: Write APIs that accept by value and move into members when cheap; this leverages copy elision and simplifies call sites.
